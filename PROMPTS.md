# PROMPTS.md

설치된 Claude Code 스킬별 프롬프트 모음집입니다.  
각 프롬프트 앞에 **언제 쓰는지**와 **어떤 결과가 나오는지** 설명을 달았습니다.

## research-paper-writer

IEEE 또는 ACM 포맷의 학술 논문 작성을 도와줍니다.  
슬래시 명령어 `/research-paper-writer`로 호출하거나, "논문 써줘"라고 말하면 자동으로 활성화됩니다.

---

### 논문 전체 초안 작성 (IEEE)

> 새 논문을 처음부터 작성할 때 사용합니다. 학술대회명과 페이지 수를 명시하면 그에 맞는 분량과 구성으로 초안을 생성합니다.

```
LLM 기반 스마트 팩토리 OEE 예측 시스템에 대한 IEEE 논문을 작성해줘.
목표 학술대회는 IEEE ICRA이고, 6페이지 분량으로 작성해줘.
```

---

### 논문 전체 초안 작성 (ACM)

> ACM 컨퍼런스 투고용 논문이 필요할 때 사용합니다. IEEE와 포맷(CCS Concepts, 단일 컬럼 Abstract 등)이 다르므로 명시적으로 ACM을 요청합니다.

```
딥러닝을 활용한 불량 감지 알고리즘에 관한 ACM 포맷 논문을 작성해줘.
기존 CNN 방식과 비교하는 실험 결과 섹션을 포함해줘.
```

---

### 연구 정보 기반 논문 초안 작성

> 연구 주제, 기여점, 데이터셋, 비교 모델이 정해진 상태에서 논문 구조를 잡을 때 사용합니다. 항목을 구체적으로 채울수록 더 정밀한 초안이 나옵니다.

```
다음 주제로 IEEE 논문 초안을 작성해줘:
- 주제: 산업용 IoT 센서 데이터를 활용한 예지보전
- 기여: Transformer 기반 이상 탐지 모델 제안
- 데이터셋: NASA CMAPSS
- 비교 기준: LSTM, Random Forest
```

---

### Abstract 개선

> 이미 작성된 논문의 Abstract를 다듬을 때 사용합니다. 단어 수 제한과 문장별 역할(배경/문제/방법/결과)을 지정하면 투고 기준에 맞게 재작성합니다.

```
작성한 논문의 Abstract를 150단어 이내로 다시 써줘.
첫 문장은 연구 배경, 마지막 두 문장은 핵심 결과와 기여를 담아줘.
```

---

### Related Work 섹션 보강

> 관련 연구 섹션이 빈약하거나 흐름이 없을 때 사용합니다. 연구 흐름을 그룹화해서 체계적으로 정리하고, 우리 연구와의 차별점도 자연스럽게 서술합니다.

```
Related Work 섹션을 보강해줘.
다음 3가지 흐름으로 그룹화해서 작성해줘:
1. 통계적 공정 관리 기반 방법
2. 머신러닝 기반 방법
3. 딥러닝 기반 방법
```

## manufacturing-expert

MES, OEE, SPC, Industry 4.0, 예지보전 등 제조 시스템 전문 조언을 제공합니다.  
슬래시 명령어 `/manufacturing-expert`로 호출하거나, 제조·품질 관련 질문을 하면 자동으로 활성화됩니다.

---

### OEE 계산

> 생산 데이터를 입력하면 가용성(Availability)·성능(Performance)·품질(Quality) 세 지표와 최종 OEE를 계산하고, 세계 수준(85%)과 비교해 개선 포인트를 제시합니다.

```
우리 충진 라인의 OEE를 계산해줘.
- 계획 생산 시간: 480분/일
- 실제 가동 시간: 420분
- 이상적 사이클 타임: 2초/개
- 실제 생산량: 11,000개
- 불량: 330개
```

---

### SPC 관리도 해석

> X̄-R 관리도에서 이상 징후가 보일 때 사용합니다. 위반 규칙(Nelson Rules)을 진단하고 즉각 조치 사항과 원인 분석 방향을 안내합니다.

```
X̄-R 관리도를 해석해줘.
UCL=512, LCL=488, 최근 20개 데이터 포인트 중 3개가 UCL을 초과했어.
어떤 조치가 필요한지 알려줘.
```

---

### 공정 능력 지수(Cpk) 해석 및 개선

> Cpk 값이 나왔을 때 현재 공정 상태를 해석하고, 규격 한계(USL/LSL) 기준으로 공정 중심 이동 또는 산포 감소 방안을 구체적으로 제안합니다.

```
Cpk = 0.95일 때 공정 능력을 어떻게 해석해야 하고,
USL=515, LSL=485 기준으로 개선 방안을 제안해줘.
```

---

### MES 아키텍처 설계

> 스마트 팩토리 구축 초기 단계에서 ISA-95 표준 기반의 MES 시스템 구조를 설계할 때 사용합니다. 레이어별 역할, ERP 연동 방식, 데이터 흐름 다이어그램을 함께 제시합니다.

```
스마트 팩토리 구축을 위한 ISA-95 기반 MES 아키텍처를 설계해줘.
ERP 연동, 실시간 생산 모니터링, 품질 데이터 수집 기능을 포함해줘.
```

---

### 설비 고장 모드 진단

> 센서 이상값이 감지됐을 때 고장 원인을 추론하고 즉각 조치 순서를 안내합니다. 진동·온도·전류 값을 함께 제공할수록 더 정확한 진단이 나옵니다.

```
진동(RMS=12.5), 온도(85°C), 전류(과전류 120%)가 감지됐어.
어떤 고장 모드가 의심되고, 즉각적인 조치는 무엇인지 알려줘.
```

---

### 가용성 계산 및 예방보전 주기 설정

> MTBF/MTTR 데이터로 설비 가용성을 계산하고, 비용 대비 최적 PM(예방보전) 주기를 산출합니다.

```
MTBF=200시간, MTTR=4시간인 설비의 가용성을 계산하고,
예방보전 주기를 어떻게 설정해야 하는지 알려줘.
```

---

### 예지보전 모델 구현

> 센서 데이터를 바탕으로 고장 확률과 잔여 수명을 예측하는 머신러닝 모델을 Python 코드로 구현합니다. 실제 현장 데이터 구조에 맞게 입력 피처를 조정해서 사용하세요.

```
다음 센서 데이터로 Random Forest 기반 고장 예측 모델을 Python으로 구현해줘:
- 입력: 진동 RMS, 온도, 전류, 압력, RPM, 누적 운전시간
- 출력: 고장 확률 및 잔여 수명(RUL) 추정
```

## proposal-writer

연구과제 제안서 및 비즈니스 제안서 작성을 도와줍니다.  
슬래시 명령어 `/proposal-writer`로 호출하거나, "제안서 써줘"라고 말하면 활성화됩니다.

---

### 국가 R&D 연구과제 제안서

> 정부 과제 신청 시 과제명·지원기관·기간·목표를 입력하면 연구 필요성, 추진 전략, 기대효과, 연구비 계획 구조를 갖춘 제안서 초안을 생성합니다.

```
다음 내용으로 국가 R&D 연구과제 제안서를 작성해줘:
- 과제명: LLM 기반 스마트제조 공정 최적화 시스템 개발
- 지원기관: 산업통상자원부
- 연구기간: 3년
- 목표: 제조 현장 OEE 15% 향상
```

---

### 솔루션 도입 비즈니스 제안서

> 고객사에 솔루션을 제안할 때 사용합니다. 문제 정의, 솔루션 개요, ROI 분석, 도입 로드맵이 포함된 제안서를 작성합니다. 고객사 업종과 수치 목표를 구체적으로 제공할수록 설득력이 높아집니다.

```
스마트팩토리 솔루션 도입 제안서를 작성해줘.
고객사는 자동차 부품 제조업체이고, 연간 불량률 3%를 1% 이하로 낮추는 것이 목표야.
ROI 분석과 도입 일정(로드맵)을 포함해줘.
```

---

### 연구비 신청서 — 연구목표 및 필요성 섹션

> 이미 연구 방향은 정해졌지만 글쓰기가 어려울 때 사용합니다. 문제·해결책·기대효과를 입력하면 심사위원 관점에서 설득력 있는 문단을 작성합니다.

```
다음 연구 목표를 바탕으로 연구비 지원 신청서의 연구목표 및 필요성 섹션을 써줘:
- 문제: 중소 제조기업의 예지보전 도입률이 5% 미만
- 해결책: 저비용 엣지 AI 기반 예지보전 플랫폼
- 기대효과: 설비 가동률 10% 향상, 유지보수 비용 30% 절감
```

---

### Executive Summary 다듬기

> 완성된 제안서의 요약본을 경영진·심사위원용으로 압축할 때 사용합니다. 핵심 수치와 기대효과를 앞쪽에 배치해 빠른 의사결정을 돕는 구조로 재작성합니다.

```
기존 제안서의 요약(Executive Summary)을 한 페이지 분량으로 다듬어줘.
의사결정권자가 읽는다고 가정하고, 핵심 수치와 기대효과를 앞쪽에 배치해줘.
```

## nxp-semiconductors-skill

NXP MCU/MPU (i.MX, Kinetis, LPC, S32), MCUXpresso SDK/IDE, FlexCAN, EdgeLock 등 임베디드 개발을 도와줍니다.  
슬래시 명령어 `/nxp-semiconductors-skill`로 호출하거나, NXP 칩·SDK 관련 질문을 하면 자동 활성화됩니다.

---

### FlexCAN 초기화 코드 작성

> i.MX RT 시리즈에서 CAN 통신을 설정할 때 사용합니다. 비트레이트, 프레임 형식, 동작 모드(폴링/인터럽트/DMA)를 지정하면 MCUXpresso SDK 기반 초기화 코드를 생성합니다.

```
i.MX RT1060에서 FlexCAN을 초기화하는 MCUXpresso SDK 코드를 작성해줘.
500kbps, 표준 프레임, 인터럽트 모드로 설정해줘.
```

---

### FreeRTOS UART 수신 태스크 구현

> LPC55S69에서 FreeRTOS와 DMA를 함께 사용해 논블로킹 UART 수신을 구현할 때 사용합니다. 태스크 우선순위, 큐 크기, DMA 채널 설정도 함께 제안합니다.

```
LPC55S69에서 FreeRTOS를 사용해서 UART 수신 태스크를 구현해줘.

```

```
S32K144의 ADC를 설정해서 12비트 분해능으로 4채널을 순차 샘플링하는
```

```
NXP EdgeLock SE050을 사용해서 ECDSA 서명 및 검증을 구현하는
```
