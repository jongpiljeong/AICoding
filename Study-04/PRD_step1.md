# PRD Step 1 — 냉장고 이미지 인식

## 개요

사용자가 냉장고 사진을 업로드하면, OpenRouter API(`nvidia/nemotron-nano-12b-v2-vl:free`)를 통해 이미지를 분석하여 식재료 목록을 추출하는 기능을 구현한다.

---

## 목표

- 사용자가 냉장고 사진을 손쉽게 업로드할 수 있는 UI 제공
- Vision 모델을 통해 이미지에서 식재료를 자동 인식
- 인식된 재료 목록을 다음 단계(레시피 생성)에 전달할 수 있는 구조화된 데이터로 반환

---

## 사용자 스토리

- 사용자는 웹 브라우저에서 냉장고 사진을 업로드하거나 카메라로 촬영할 수 있다.
- 업로드 후 "재료 인식하기" 버튼을 누르면 AI가 이미지를 분석한다.
- 분석이 완료되면 인식된 재료 목록이 화면에 표시된다.
- 사용자는 인식된 재료를 수동으로 추가·삭제·수정할 수 있다.
- "레시피 추천받기" 버튼으로 Step 2로 진행할 수 있다.

---

## 기능 요구사항

### F1. 이미지 업로드
| ID | 요구사항 |
|----|----------|
| F1-1 | 파일 선택(drag & drop 포함) 방식으로 이미지 업로드 지원 |
| F1-2 | 지원 형식: JPG, PNG, WEBP (최대 10MB) |
| F1-3 | 업로드된 이미지를 미리보기로 표시 |
| F1-4 | 모바일 환경에서 카메라 촬영 직접 연결 지원 (`capture="environment"`) |

### F2. AI 이미지 분석
| ID | 요구사항 |
|----|----------|
| F2-1 | 이미지를 base64로 인코딩하여 OpenRouter API에 전송 |
| F2-2 | 모델: `nvidia/nemotron-nano-12b-v2-vl:free` |
| F2-3 | 프롬프트: 이미지에서 식재료 이름만 JSON 배열로 추출 요청 |
| F2-4 | API 호출 중 로딩 스피너 표시 |
| F2-5 | API 오류 시 에러 메시지를 사용자에게 노출하고 재시도 버튼 제공 |

### F3. 재료 목록 표시 및 편집
| ID | 요구사항 |
|----|----------|
| F3-1 | 인식된 재료를 태그(chip) 형태로 목록 표시 |
| F3-2 | 각 태그에 삭제(X) 버튼 제공 |
| F3-3 | 텍스트 입력으로 재료 직접 추가 가능 |
| F3-4 | 최종 재료 목록을 로컬 상태(state)에 저장 |

## API 명세

### POST /api/analyze-image

**Request**
```json
{
  "image": "data:image/jpeg;base64,/9j/4AAQ..."
}
```

**Response (성공)**
```json
{
  "ingredients": ["달걀", "우유", "당근", "양파", "치즈"]
}
```

**Response (실패)**
```json
{
  "error": "이미지를 인식할 수 없습니다."
}
```

**OpenRouter 호출 프롬프트**
```
이 냉장고 사진에서 보이는 모든 식재료를 찾아서 JSON 배열 형식으로만 응답해주세요.
형식: {"ingredients": ["재료1", "재료2", ...]}
설명 없이 JSON만 출력하세요.
```

│  │                                   │  │

- 재료 태그는 색상으로 구분 (자동 인식 vs 수동 추가)

| 항목 | 선택 |

---

## 비기능 요구사항

- API 응답 시간: 30초 이내 (타임아웃 설정)
- 이미지 크기: 업로드 전 클라이언트에서 리사이즈 (최대 1024px)
- API 키는 서버 사이드(`/api`)에서만 사용, 클라이언트에 절대 노출 금지
- Rate Limit 발생 시 사용자에게 "잠시 후 다시 시도해주세요" 안내
