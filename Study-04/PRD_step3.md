# PRD Step 3 — 사용자 프로필 및 레시피 저장

## 개요

사용자 프로필을 생성하여 Step 2에서 생성된 레시피를 저장·관리할 수 있는 개인화 기능을 구현한다. 로그인 없이 사용 가능한 로컬 프로필과 향후 확장을 위한 계정 구조를 설계한다.

---

## 목표

- 회원가입/로그인 없이 즉시 사용 가능한 로컬 프로필 제공
- 저장된 레시피를 언제든지 조회·관리할 수 있는 마이페이지 구현
- 식단 선호도, 알레르기 정보 등 개인 설정을 프로필에 저장하여 레시피 추천에 반영
- 향후 계정 연동(소셜 로그인) 확장을 고려한 데이터 구조 설계

---

## 사용자 스토리

- 사용자는 처음 방문 시 닉네임과 기본 식단 선호도를 입력하여 프로필을 만들 수 있다.
- Step 2에서 "저장하기"를 누르면 해당 레시피가 프로필의 저장 목록에 추가된다.
- 마이페이지에서 저장된 레시피를 목록으로 확인하고 상세 조회할 수 있다.
- 저장된 레시피에 별점과 메모를 남길 수 있다.
- 사용자는 프로필 설정을 언제든지 수정할 수 있다.
- 저장된 레시피를 삭제하거나 즐겨찾기로 분류할 수 있다.

## 기능 요구사항

### F1. 프로필 생성 및 관리
| ID | 요구사항 |
|----|----------|
| F1-1 | 첫 방문 시 프로필 설정 온보딩 화면 표시 (건너뛰기 가능) |
| F1-2 | 프로필 정보: 닉네임, 프로필 이모지/색상, 식단 선호도, 알레르기 재료 |
| F1-3 | 식단 선호도: 일반 / 채식 / 비건 / 저칼로리 / 글루텐 프리 (다중 선택) |
| F1-4 | 알레르기 재료를 텍스트로 직접 입력 (예: 견과류, 유제품) |
| F1-5 | 프로필 데이터는 브라우저 localStorage에 저장 |
| F1-6 | 프로필 설정값은 Step 2 레시피 생성 옵션에 자동 반영 |

### F2. 레시피 저장
| ID | 요구사항 |
|----|----------|
| F2-1 | Step 2의 "저장하기" 버튼 클릭 시 레시피 데이터를 프로필에 저장 |
| F2-2 | 저장 시 메타데이터 자동 기록: 저장 일시, 사용한 재료 목록, 원본 이미지(썸네일) |
| F2-3 | 동일 레시피 중복 저장 방지 (제목 기준 확인 후 덮어쓰기 여부 사용자 확인) |
| F2-4 | 저장 성공 시 토스트 알림 표시 |
| F2-5 | 저장된 레시피 최대 100개 제한 (초과 시 오래된 항목 삭제 안내) |

### F3. 저장된 레시피 목록 (마이페이지)
| ID | 요구사항 |
|----|----------|
| F3-1 | 저장된 레시피를 카드 그리드로 표시 |
| F3-2 | 정렬 옵션: 최신순 / 별점순 / 요리 시간순 |
| F3-3 | 필터: 전체 / 즐겨찾기 / 난이도 / 식단 유형 |
| F3-4 | 검색: 요리명 또는 재료명으로 검색 |
| F3-5 | 카드 클릭 시 레시피 상세 페이지 이동 |

### F4. 레시피 상세 및 평가
| ID | 요구사항 |
|----|----------|
| F4-1 | 저장된 레시피 전체 내용 표시 (Step 2 카드와 동일 레이아웃) |
| F4-2 | 별점 입력: 1~5점 (클릭으로 선택) |
| F4-3 | 메모 입력: 자유 텍스트 (최대 500자) |
| F4-4 | 즐겨찾기 토글 버튼 (★) |
| F4-5 | 레시피 삭제 버튼 (확인 다이얼로그 포함) |
| F4-6 | "이 재료로 다시 추천받기" 버튼 → Step 2로 재료 목록 전달 |

## 데이터 구조

### localStorage 스키마

```typescript
// 사용자 프로필
interface UserProfile {
  id: string;              // 랜덤 UUID
  nickname: string;
  avatar: string;          // 이모지 (예: "👨‍🍳")
  avatarColor: string;     // 배경 색상 hex
  dietPreferences: string[];   // ["일반", "저칼로리"]
  allergies: string[];         // ["견과류", "유제품"]
  createdAt: string;       // ISO 8601
  updatedAt: string;
}

// 저장된 레시피
interface SavedRecipe {
  id: string;              // 레시피 UUID
  title: string;
  time: number;            // 분
  difficulty: "쉬움" | "보통" | "어려움";
  servings: number;
  usedIngredients: string[];
  extraIngredients: string[];
  steps: string[];
  thumbnail: string | null;    // base64 썸네일 (원본 냉장고 사진 축소)
  savedAt: string;         // ISO 8601
  rating: number | null;   // 1~5, null = 미평가
  memo: string;
  isFavorite: boolean;
  sourceIngredients: string[];  // Step 1에서 인식된 전체 재료
}

// localStorage 키 구조
// "fridgeRecipe_profile"   → UserProfile
// "fridgeRecipe_recipes"   → SavedRecipe[]
```

## 기술 스택

| 항목 | 선택 |
|------|------|
| 프레임워크 | Next.js 14 (App Router) |
| 스타일 | Tailwind CSS |
| 상태 관리 | React Context + localStorage hook |
| 데이터 저장 | localStorage (클라이언트 전용) |
| 라우팅 | `/profile` — 프로필 설정, `/my-recipes` — 저장 목록, `/my-recipes/[id]` — 상세 |

---

## 비기능 요구사항

- localStorage 저장 용량 한도: 5MB 기준으로 썸네일 이미지 크기 관리 (최대 50KB/장)
- 프로필 없이도 레시피 저장 가능 (익명 프로필 자동 생성)
- 브라우저 localStorage 비활성화 환경에서는 세션 내 메모리 저장으로 fallback
- 향후 서버 동기화(DB 연동) 확장을 위해 데이터 구조를 REST API 호환 형태로 설계

## 전체 플로우 연결

```
[Step 1: 이미지 업로드·인식]
        ↓ 재료 목록
[Step 2: 레시피 생성]
        ↓ 저장하기
[Step 3: 마이페이지 저장]
        ↓ "이 재료로 다시 추천받기"
[Step 2: 레시피 재생성] (순환)
```
