## 1. Executive Summary

PDF 문서 요약 웹 앱은 사용자가 PDF 파일을 브라우저에 업로드하면 AI가 자동으로 내용을 요약해주는 순수 클라이언트 사이드(Client-Side Only) 서비스입니다. 단일 HTML 파일(`index_pdf.html`)로 구성되며, 서버 설치·배포 없이 파일 시스템에서 직접 실행(`file://` 프로토콜)할 수 있습니다. PDF.js(텍스트 추출)와 OpenRouter API(AI 요약)를 CDN 및 외부 API로만 활용하여 개발·운영 비용을 최소화하면서도 실용적인 문서 요약 경험을 제공합니다.

---

## 2. Problem Statement

### 해결하는 문제
- 긴 PDF 문서(보고서, 논문, 계약서 등)를 처음부터 읽는 데 많은 시간이 소요됨
- 기존 AI 요약 서비스는 회원가입·서버 설치·비용 결제 등 진입 장벽이 존재함
- 문서 내용을 외부 서버에 업로드하는 것에 대한 개인정보·보안 우려

### 대상 사용자
- 연구자, 학생: 논문·보고서를 빠르게 파악해야 하는 사람
- 직장인: 계약서, 제안서, 공문 등을 빠르게 검토해야 하는 사람
- 개인 개발자·테크 사용자: 로컬 환경에서 가볍게 쓸 수 있는 도구가 필요한 사람

### 현재 Pain Point 및 비즈니스 영향
- 수십 페이지 PDF를 검토하는 데 평균 30분~2시간 소요 → 생산성 저하
- 기존 SaaS 요약 도구는 월 구독료 발생 또는 페이지 수 제한 존재
- 파일을 외부 서버에 올리지 않아도 되는 로컬 처리 방식에 대한 수요 증가

## 3. Goals & Success Metrics

### 비즈니스 목표
- 서버 인프라 비용 0원으로 동작하는 경량 도구 제공
- 단일 HTML 파일로 즉시 배포 가능한 구조 유지

### 사용자 목표
- PDF 업로드부터 요약 결과 확인까지 **60초 이내** 완료
- 요약 정확도에 대한 사용자 체감 만족도 **4점/5점 이상** (정성 평가 기준)

### KPIs
| 지표 | 목표값 | 측정 방법 |
|------|--------|-----------|
| 텍스트 추출 성공률 | 95% 이상 (텍스트 레이어 포함 PDF 기준) | 브라우저 콘솔 로그 |
| 전체 요약 소요 시간 | 20MB PDF 기준 60초 이내 | 타임스탬프 측정 |
| API 오류율 | 5% 미만 | 오류 메시지 노출 횟수 |
| localStorage 히스토리 저장 성공률 | 100% | 저장 후 조회 확인 |

### 범위 외 목표 (Out of Scope)
- 이미지 PDF(스캔본) OCR 처리 — Phase 2 검토
- 사용자 계정 및 클라우드 동기화
- 다중 파일 동시 처리
- PDF 편집 또는 주석 기능

## 4. User Personas

### Persona A — 대학원생 "김연구"
- **역할**: 공학 석사과정, 주 3~5편 논문 리뷰
- **목표**: 논문의 핵심 기여점을 5분 안에 파악
- **불만**: Zotero나 Notion AI는 유료, ChatGPT에 붙여넣기는 글자 수 제한 존재
- **기술 수준**: 중급 (Git, 터미널 사용 가능, 서버 설치는 부담)

### Persona B — 직장인 "이기획"
- **역할**: 마케팅팀 과장, 주간 보고서·제안서 검토 업무
- **목표**: 15~30페이지 제안서를 경영진 보고 전 빠르게 요점 정리
- **불만**: 회사 보안정책상 외부 SaaS에 내부 문서 업로드 불가
- **기술 수준**: 일반 (설치 없이 HTML 파일 열기만 가능)

## 6. Feature Specifications

### 6.1 PDF 업로드 컴포넌트

**구현 사양:**
```
- HTML <input type="file" accept=".pdf"> (숨김 처리)
- 드롭존 div에 dragover, dragleave, drop 이벤트 핸들러
- 파일 유효성 검사 순서:
  1. MIME type === 'application/pdf' 또는 확장자 === '.pdf'
  2. file.size <= 20 * 1024 * 1024 (20MB)
  3. 통과 시 텍스트 추출 단계로 진행
```

**엣지 케이스:**
- 여러 파일을 동시에 드롭 → 첫 번째 파일만 처리, 나머지 무시
- 동일 파일 재업로드 → 정상 처리 (캐시 없이 재추출)
- 파일명에 한글·특수문자 포함 → 정상 표시 (URL 인코딩 불필요)

### 6.2 텍스트 추출 컴포넌트 (PDF.js)

**구현 사양:**
```javascript
// PDF.js 초기화
pdfjsLib.GlobalWorkerOptions.workerSrc = 
  'https://cdnjs.cloudflare.com/ajax/libs/pdf.js/3.11.174/pdf.worker.min.js';

// 추출 로직 (페이지별 순차 처리)
async function extractText(file) {
  const arrayBuffer = await file.arrayBuffer();
  const pdf = await pdfjsLib.getDocument({ data: arrayBuffer }).promise;
  let fullText = '';
  
  for (let i = 1; i <= pdf.numPages; i++) {
    const page = await pdf.getPage(i);
    const content = await page.getTextContent();
    const pageText = content.items.map(item => item.str).join(' ');
    fullText += pageText + '\n';
    updateProgress(i, pdf.numPages); // 진행률 업데이트
  }
  return fullText;
}
```

**스캔 PDF 감지:**
- 추출된 전체 텍스트 길이 < 50자이면 스캔 PDF로 판정
- 에러 상태로 전환 후 사용자 안내

**토큰 제한 대응:**
- 추출된 텍스트를 `text.slice(0, 12000)` 문자 기준으로 트리밍
- 트리밍 발생 시 UI에 "문서가 길어 앞부분(약 12,000자)만 요약합니다." 안내
- 12,000자 ≈ 약 3,000~4,000 토큰 (한국어 기준)

### 6.3 AI 요약 컴포넌트

**요약 유형별 프롬프트:**

```
[전체 요약]
System: 당신은 문서 요약 전문가입니다. 주어진 텍스트를 {언어}로 요약하세요.
User: 다음 텍스트를 3~5개 단락으로 구성된 전체 요약으로 작성해주세요:\n\n{text}

[핵심 포인트]
System: 당신은 문서 요약 전문가입니다. 주어진 텍스트를 {언어}로 요약하세요.
User: 다음 텍스트에서 핵심 포인트를 불릿 포인트(•) 5~7개로 추출해주세요:\n\n{text}

[한 줄 요약]
System: 당신은 문서 요약 전문가입니다. 주어진 텍스트를 {언어}로 요약하세요.
User: 다음 텍스트를 핵심 내용 중심으로 1~2문장으로 요약해주세요:\n\n{text}
```

**API 요청 구조:**
```javascript
const response = await fetch('https://openrouter.ai/api/v1/chat/completions', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${apiKey}`,
    'Content-Type': 'application/json',
    'HTTP-Referer': 'file://',       // OpenRouter 정책 준수
    'X-Title': 'PDF Summary App'
  },
  body: JSON.stringify({
    model: selectedModel,            // 모델 선택 (섹션 7 참조)
    messages: [
      { role: 'system', content: systemPrompt },
      { role: 'user', content: userPrompt }
    ],
    max_tokens: 1024,
    temperature: 0.3                 // 일관성 있는 요약을 위해 낮은 값 사용
  })
});
```

### 6.5 히스토리 컴포넌트

**localStorage 데이터 구조:**

// Key: 'pdf_summary_history'
// Value: JSON Array (최대 5개)
[
  {
    id: "uuid-v4",
    fileName: "sample.pdf",
    summaryType: "전체 요약",

- 저장 실패(용량 초과) 시 가장 오래된 항목부터 제거 후 재시도

| 3 | `deepseek/deepseek-r1-distill-llama-70b:free` | 추론 능력 강함 | 32K 토큰 |
| 4 | `microsoft/phi-3-medium-128k-instruct:free` | 경량, 빠른 처리 | 128K 토큰 |
| 5 | `mistralai/mistral-7b-instruct:free` | 안정적, 영어 특화 | 32K 토큰 |

> 선택 전략: 기본값은 1순위(`gemini-2.0-flash-exp:free`). 모델 드롭다운으로 사용자가 변경 가능. 무료 모델은 RPM(분당 요청수) 제한이 있으므로 요청 실패 시 다음 순위 모델로 자동 폴백하는 로직 구현 권장.

### 7.3 토큰 제한 전략

```
문서 길이 처리 방식:
1. 추출 텍스트 전처리
   - 연속 공백·줄바꿈 정규화: text.replace(/\s+/g, ' ').trim()
   - 특수문자 제거(선택): 불필요한 제어문자 제거

2. 청크 분할 기준
   - 12,000자 이하: 단일 요청으로 처리
   - 12,001자 ~ 36,000자: 3등분 후 각 청크 요약 → 최종 통합 요약 (Phase 2)
   - 현재(Phase 1): 12,000자 초과 시 앞부분만 처리 + 사용자 안내

3. max_tokens 설정
   - 전체 요약: 800 tokens
   - 핵심 포인트: 600 tokens
   - 한 줄 요약: 200 tokens
```

### 7.4 API Key 보안

- API Key는 HTML 파일 내 하드코딩 금지
- 앱 최초 실행 시 입력 폼 표시 → `sessionStorage.setItem('openrouter_key', key)` 저장
- 페이지 새로고침/종료 시 자동 삭제 (sessionStorage 특성 활용)
- "API Key 저장" 체크박스 선택 시에만 `localStorage`에 암호화 없이 저장 (보안 경고 문구 함께 표시)

### 7.5 데이터 프라이버시

- PDF 파일은 서버로 전송되지 않음 (브라우저 메모리 내에서만 처리)
- 추출된 텍스트는 OpenRouter API로 전송됨 — UI에 명시
- localStorage 저장 데이터는 해당 브라우저·도메인에만 국한

## 11. Risks & Mitigations

| 리스크 | 발생 가능성 | 영향도 | 완화 전략 |
|--------|-----------|--------|----------|
| OpenRouter 무료 모델 서비스 중단 | 중 | 높음 | 5개 모델 폴백 목록 유지, 사용자에게 모델 선택 권한 부여 |
| PDF.js CDN 접근 불가 | 낮음 | 높음 | 로드 실패 시 명확한 에러 메시지 표시, 대안 CDN URL 주석으로 제공 |
| 스캔 PDF 오탐지 | 중 | 중간 | 임계값 50자 조정 가능하도록 상수로 정의 (`MIN_TEXT_LENGTH = 50`) |
| localStorage 용량 초과 | 중 | 낮음 | 저장 실패 시 자동 정리 로직, 원문 텍스트 저장 길이 제한 (3,000자) |
| OpenRouter API Rate Limit (무료 모델) | 높음 | 중간 | 에러 429 시 UI에 대기 시간 표시 + 자동 폴백 모델 전환 |
| file:// 프로토콜에서 API CORS 오류 | 낮음 | 높음 | OpenRouter는 file:// 허용 확인 필요; 사전 테스트 필수 |
| 대용량 PDF 브라우저 메모리 부족 | 중 | 중간 | 20MB 파일 크기 제한으로 완화; 페이지별 순차 처리로 메모리 관리 |

## 12. Open Questions

1. **CORS 정책**: OpenRouter API가 `file://` 프로토콜 Origin에서의 요청을 허용하는지 사전 실제 테스트 필요. 불허 시 `null` Origin 헤더 처리 방법 검토.

2. **API Key localStorage 저장**: "저장" 선택 시 평문 저장에 대한 보안 경고 수준을 어느 정도로 설정할지 결정 필요.

3. **다국어 프롬프트**: 언어 선택 시 System Prompt의 언어도 동일하게 변경할지, 아니면 한국어로 고정할지 결정 필요.

4. **히스토리 원문 저장 범위**: 원문 텍스트 전체(최대 12,000자)를 localStorage에 저장하면 5개 항목 기준 최대 ~300KB 소요. 용량 대비 편의성 트레이드오프 결정 필요.

5. **모바일 지원**: 현재 명세는 데스크톱 중심. 모바일 브라우저 파일 선택 UX 및 레이아웃 반응형 지원 여부 결정 필요.
